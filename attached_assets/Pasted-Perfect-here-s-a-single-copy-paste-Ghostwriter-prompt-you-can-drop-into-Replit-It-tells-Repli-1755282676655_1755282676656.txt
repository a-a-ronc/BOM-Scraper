### 3) Backend specifics

* **settings.py**: load `OPENAI_API_KEY`, `DATABASE_URL` (sqlite default), CORS.
* **models.py (SQLModel)**:

  * `User(id, email, password_hash, created_at)`
  * `Project(id, name, customer, address, drawing_no, revision, metadata_json, created_by, created_at)`
  * `ProjectFile(id, project_id, filename, page_count, parsed_json, ocr_json, created_at)`
  * `BomItem(id, project_id, family, part_code, description, uom, qty, vendor, calc_inputs_json)`
  * `AgentLog(id, project_id, stage, message, details_json, created_at)`
* **routers**:

  * `POST /auth/register`, `POST /auth/login`
  * `POST /projects` → multipart upload (one or more PDFs). If it’s the first file, auto‑create Project by parsing title block; otherwise attach to existing via query `project_id`.
  * `POST /projects/{id}/parse` → run `parser_pipeline.run(id)` (idempotent).
  * `GET /projects`, `GET /projects/{id}`
  * `POST /projects/{id}/bom` → body: `{ vendors: ["interlake","hannibal","stow"] }`
  * `GET /projects/{id}/bom?vendor=interlake` (JSON lines) and `GET /projects/{id}/bom/export?fmt=csv|xlsx`
  * `GET /healthz`
* **services/parser\_pipeline.py**:

  * Stage A: `text_extract.extract(pdf_path)` with PyMuPDF/pdfplumber
  * Stage B: `ocr_extract.ocr_images(pdf_path)` with PaddleOCR for any rasterized pages
  * Stage C: `dims_extract.parse_dims(text)` → normalize feet/inches (e.g., `6'-0"`, `4'-4"`) and loads (`1,000 lbs`)
  * Stage D: `normalize.merge_conflicts(a,b)`; when conflicts remain, call `agent_openai.resolve_conflict(context)` to choose + explain; persist an `AgentLog`.
  * Return a normalized dict:

    ```json
    {
      "customer":"Falcon Fulfillment",
      "address":"1065 Conestoga Pkwy, Shepherdsville, KY 40165",
      "drawing_no":"D-241254-R-180",
      "views":["TOP VIEW","ELEVATION VIEW"],
      "pallet":{"size":"40x48x52","unit_load_lb":1000},
      "clear_height_in":72,
      "lowest_truss_in":424
    }
    ```

    (values derived from the examples above)
* **services/bom/rules\_engine.py**:

  * Load YAML → produce context variables (bays, levels, beam\_length\_in, etc.)
  * Evaluate with `numexpr` in a safe environment
  * Apply overrides (e.g., anchors per footplate = 4 when heavy/seismic)
* **services/bom/vendor\_map.py**:

  * Map generic families → vendor SKUs/descriptions by template
* **services/bom/bom\_builder.py**:

  * Compute frames = `bays + 1`; beams = `bays * levels * 2`; wire\_decks = `bays * levels * ceil(beam_length_in / deck_module_width_in)`; anchors = `uprights * anchors_per_footplate`.
* **services/agent/agent\_openai.py**:

  * Thin wrapper using `openai` (or `openai>=1.0` SDK) with model `gpt-4o-mini` by default; expose `resolve_conflict(context: dict) -> dict`.

### 4) YAML samples

* **/server/rules/default\_rules.yaml**

```yaml
defaults:
  anchors_per_footplate: 2
  deck_module_width_in: 48
  row_spacer_spacing_ft: 8
overrides:
  heavy_load:
    triggers:
      - pallet_weight_lb >= 1500
      - seismic_zone >= 3
    set:
      anchors_per_footplate: 4
calculations:
  frames: "bays + 1"
  beams: "bays * levels * 2"
  wire_decks: "bays * levels * ceil(beam_length_in / deck_module_width_in)"
  anchors: "uprights * anchors_per_footplate"
```

* **/server/vendors/default\_vendor\_map.yaml**
